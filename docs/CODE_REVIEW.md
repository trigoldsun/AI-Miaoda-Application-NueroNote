# 飞书消息卡片
# 通知：NueroNote 代码安全审查完成

## 🔍 审查结果：修复了 20+ 个问题

### 🔴 严重（已修复）

**1. 加密引擎 — 删除了不安全的后备加密方案**
- 问题：Fallback 用了自创的 HMAC-SHA256，不是真正的 AES-GCM
- 修复：移除不安全后备，不存在时直接报错（`pip install cryptography`）
- 影响：用户数据安全

**2. 服务端 — 关闭 Debug 模式**
- 问题：`app.run(debug=True)` → Werkzeug debugger 可远程代码执行
- 修复：`debug=False`（生产安全要求）
- 影响：服务器安全

**3. 服务端 — JWT Secret 重启后丢失**
- 问题：`secrets.token_hex(32)` 在每次启动时生成新值
- 影响：重启后所有用户 Token 失效
- 修复：从环境变量 `FLUX_JWT_SECRET` 读取，启动时警告未设置

**4. 服务端 — 缺少请求体大小限制**
- 问题：无 `MAX_CONTENT_LENGTH` → 恶意客户端可发无限大请求
- 修复：限制 1MB，并返回 413 错误

**5. 服务端 — 空搜索返回全量数据**
- 问题：`search('')` 返回所有文档，暴露数据
- 修复：空查询返回 `[]`

### 🟡 中等（已修复）

**6. 加密引擎 — Argon2 后备实现错误**
- 问题：截断 Argon2 哈希输出作为密钥，破坏安全性
- 修复：统一使用 PBKDF2-SHA256（10万次迭代），删除有问题的后备

**7. 服务端 — 无账户锁定机制**
- 问题：暴力破解无限制
- 修复：5次失败后锁定15分钟

**8. 服务端 — 无 IP 限流**
- 问题：注册/登录/Sync 接口无速率限制
- 修复：滑动窗口限流（注册10次/小时，登录20次/5分钟，同步60次/分钟）

**9. 服务端 — 无存储配额**
- 问题：用户可上传任意大文件
- 修复：Free 512MB / Pro 10GB / Team 100GB，配额超限返回 507

**10. 服务端 — 乐观锁竞态条件**
- 问题：版本检查和写入之间存在 TOCTOU 窗口
- 修复：使用 `UPDATE ... WHERE version = ?` 原子操作

### 🟢 轻微（已修复）

**11. 服务端 — 无审计日志**
- 修复：所有写操作（注册/登录/登出/同步/升级）记录审计日志

**12. 服务端 — 错误信息泄露用户存在**
- 问题：登录时邮箱不存在返回 404
- 修复：统一返回 401，固定延迟 200ms

**13. 服务端 — 无 JWT 过期字段**
- 修复：添加 `iat`（签发时间）和 `jti`（唯一 ID，防重放）

**14. 服务端 — 缺少安全响应头**
- 修复：`X-Content-Type-Options`、`X-Frame-Options`

**15. 加密 — 常量时间比较**
- 问题：密钥验证用 `==` 可能被时序攻击
- 修复：改用 `hmac.compare_digest`

**16. 数据模型 — daily_notes 字段名不一致**
- 问题：`_seal()` 用 `daily_notes`，`create()` 用 `daily`
- 修复：统一用 `daily`

**17. 数据模型 — 缺少 `seal()` 方法暴露**
- 问题：用户需手动调用 `_seal()` 私有方法
- 修复：添加公开 `seal()` 方法

**18. Vault — 缺少 from_dict 工厂方法**
- 问题：只能用 dict 解锁
- 修复：添加 `EncryptedVault.from_dict()`

**19. 服务端 — SQLite 并发配置**
- 修复：WAL 模式 + `PRAGMA foreign_keys=ON` + `isolation_level=None`

**20. 健康检查端点 — 缺少 `/api/v1/security.txt`**
- 修复：添加 RFC 9116 安全联系页面
